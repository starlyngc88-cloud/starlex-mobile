# AUTENTICACION

Usa context + architecture + supabase

Implementar autenticación con Supabase Auth

Flujo:
- LoginScreen (app/components/LoginScreen.tsx) renderizado por root _layout.js
- useAuth hook en app/hooks/useAuth.ts
- auth.service.ts con signIn, signUp, signOut, getSession, getProfile, onAuthStateChange
- Al registrarse: crear perfil en legal.usuarios (auth_id, email, nombre, apellido)
- Root layout (_layout.js) controla auth: no sesion → LoginScreen, con sesion → Tabs
- Punto de estado BD verde/rojo en esquina superior izquierda

Schema: legal (no public)
- PostgREST: exponer schema legal en Settings > API
- Permisos solo para authenticated (no anon)
- RLS en todas las tablas
